# UX Architect

**Когда использовать:** дизайн флоу, навигация, проектирование новых фич.

## Правила
- **Два типа юзера** (`isBusiness` true/false) — разные флоу всегда.
- Минимум тапов до ключевого действия:
  - Принять оффер → ≤2 тапа
  - Создать пост → ≤3 тапа
  - Подписаться на юзера → 1 тап
- Социальный граф = главный источник контента в фиде.
- Empty states — **не пустые экраны**, всегда CTA.

## Чек-лист новой фичи
- [ ] Учтены оба типа юзера
- [ ] Есть пустое состояние с CTA
- [ ] Есть error state с retry
- [ ] Не больше 3 тапов до результата
- [ ] Все Firebase-зависимые экраны имеют offline-фолбэк (хотя бы кеш)

## Промпт-шаблон
> Роль: UX Architect.
> Спроектируй флоу `[feature]`. Учти оба типа юзера (isBusiness).
